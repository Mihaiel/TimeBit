Projektinformationen

Stand: 12.04.2025

Regeln:

  • Erst syncrhonisieren (pullen) dann hochladen (pushen).
  • Der Code ist ausführlich zu kommentieren.
  • Für Funktionen klare und einheitliche Namen verwenden.
  • Kein CSS oder JavaScript direkt im HTML.
  • Keine persönlichen Daten im Code. Vertrauliche Daten werden immer aus .emv rausgelesen.
  • Die HTML Seiten dürfen nur main.css und die entsprechende .css der dazugehörigen HTML Seite verwenden. (z.B: Projects darf nur main.css und projects.css verwenden)
  • Die Datei main.css dient als zentrale Vorlage und darf nur geändert werden, wenn Stil-Elemente betroffen sind, die auf mehreren Seiten verwendet werden.
  • Das Frontend soll GENAU so ausschauen wie es in Figma entwurfen wurde.

Projects

2. API-Routen für Projekte im Backend erstellen
